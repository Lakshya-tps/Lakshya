Use case overview

How the authentication path works

  • Register identity: capture a face, score quality, and create the local identity hash.
  • Store hash: when blockchain is ready, the identity hash is written and later checked again.
  • Authenticate user: a fresh capture is matched against the saved profile before verification state is shown.
  • Retrieve hash: chain state is pulled during verification to detect missing or mismatched records.

Step 1

Capture face

The webcam frame is taken locally and checked for blur, lighting, and framing quality.

Step 2

Match and verify

The fresh face encoding is compared with the stored identity profile before access is accepted.

Step 3

Check blockchain state

If the chain is online, the stored hash is retrieved and compared against the local record.
